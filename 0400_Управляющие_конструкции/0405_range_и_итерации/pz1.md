## Практическое задание: цикл `for` и функция `range()` в Python

### Цель

Закрепить понимание работы цикла `for` и функции `range()` в Python, научиться управлять диапазонами перебора, шагом итерации и условиями внутри цикла. Отработать навыки написания простых алгоритмов обработки числовых последовательностей.

### Задание (шаги)

1. Напишите программу, которая выводит все числа от 1 до 50 включительно, используя `range()`.

2. Измените программу так, чтобы она выводила только чётные числа от 2 до 50.

3. Добавьте вычисление суммы всех чисел от 1 до 100.

4. Напишите программу, которая выводит числа от 100 до 1 в обратном порядке.

5. Создайте программу, которая:
   - перебирает числа от 1 до 30,
   - выводит только числа, которые делятся на 3 без остатка,
   - одновременно считает их количество.

6. Дополнительно (уровень выше):
   - с помощью `range()` создайте список квадратов чисел от 1 до 10,
   - выведите полученный список.

### Подсказки по ключевым частям

- `range(n)` — от 0 до n-1

- `range(a, b)` — от a до b-1

- `range(a, b, step)` — шаг итерации

- Для обратного прохода используйте отрицательный шаг: `range(100, 0, -1)`

- Проверка делимости:

  ```python
  if number % 3 == 0:
  ```

- Суммирование:
  - используйте переменную-накопитель, например `total = 0`

- Генерация квадратов:
  - `i * i` внутри цикла

### Что проверить перед отправкой (чек-лист)

- [ ] Используется `range()` вместо ручного перечисления чисел
- [ ] Цикл `for` корректно проходит по диапазону
- [ ] Шаг (`step`) применён там, где это требуется
- [ ] Условия (`if`) написаны без логических ошибок
- [ ] Суммы и счётчики инициализированы до цикла
- [ ] Программа не содержит лишнего кода
- [ ] Результат соответствует каждому пункту задания

### Советы по улучшению работы

- Не смешивайте логику вывода и вычислений без необходимости — сначала счёт, затем вывод.
- Проверяйте диапазоны `range()` отдельно, чтобы избежать ошибок "на 1".
- Используйте осмысленные имена переменных (`sum_numbers`, `count_div3`), а не `a`, `b`.
- Если цикл становится перегруженным, разбивайте задачу на части: сначала фильтрация, затем подсчёт.
- Тренируйтесь мысленно прогонять цикл: это помогает быстрее находить ошибки.
